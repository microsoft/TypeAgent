# Style guide

Conventions for wiki Markdown so pages render cleanly in DocFX and read
consistently. These apply to **curated** wiki pages; the package READMEs staged
in from `ts/packages` follow the conventions of their own canonical home.

## Page structure

- Start every page with a single `#` H1 that matches its purpose. DocFX uses it
  as the page title and breadcrumb.
- Use sentence case for headings ("Add a package", not "Add A Package").
- Lead with a short paragraph that says what the page is and who it is for.
- Prefer short sections with descriptive `##`/`###` headings over long prose.

## Links

- **Wiki-to-wiki:** link to the target `.md` file with a relative path, e.g.
  `[Dispatcher](../architecture/core/dispatcher.md)`. DocFX rewrites and validates
  these; a broken relative link is a build warning.
- **To source code or files outside the docset:** use an absolute GitHub URL,
  e.g. `https://github.com/microsoft/TypeAgent/blob/main/ts/packages/...`.
  Relative links to source (`./src/...`) will not resolve to a wiki page.
- Use descriptive link text, not "here" or a bare URL.

## Markdown that DocFX likes

- **Tables** render well — use them for option lists and comparisons.
- **Fenced code blocks** with a language hint (` ```ts `, ` ```bash `,
  ` ```json `) get syntax highlighting.
- **Alerts:** DocFX supports GitHub-style and `> [!NOTE]` / `> [!TIP]` /
  `> [!WARNING]` blockquote alerts. Plain `>` blockquotes also work.
- **Includes:** reuse a snippet with `[!INCLUDE [label](./path.md)]`.
- Keep raw HTML to a minimum; prefer Markdown so content stays portable.

## Images

- Put wiki-owned images under `ts/docs/media/` and reference them with a
  relative path. Architecture-doc images continue to live in
  `ts/docs/architecture/images/`.
- Always provide alt text.

## Tone

- TypeAgent is **sample code** exploring an architecture — frame docs that way,
  not as product documentation.
- Be concise and concrete. Prefer an example or a table over a paragraph.

## What not to do

- Don't hand-edit generated files (`*/toc.yml` with an `# <auto-generated>`
  header, or any staged `README.AUTOGEN.md` / `overview.md` / `generated.md`
  under `packages/` or `agents/`).
- Don't duplicate content across pages — link to the canonical source instead.
- Don't add a second copy of a package's docs inside the wiki; edit the
  package's `README.md` under `ts/packages/**`.
