# DSL Future Ideas

This folder tracks exploratory DSL surface ideas that are not part of
the current contract.

Use one file per idea.

Suggested template (same as `ir/future/`):

- Title
- Status
- Context
- Candidate option(s)
- Why consider this
- Risks and costs
- Open questions
- Non-goal

When an idea is accepted, fold the final behavior into
`docs/design/workflowSystem/dsl/dsl-v0.1.md` (or the active DSL spec),
then mark or remove the future note.
